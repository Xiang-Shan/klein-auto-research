The Method

If you know principal component analysis, you are two ideas away from a denoising autoencoder. This section teaches both ideas, then the two tabular-specific tricks (swap noise, RankGauss) that make the method work on policy data, then the honest story of where it came from — including the part of that story that predicts it may not work here.

Idea 1 — an autoencoder is nonlinear PCA

PCA finds a low-dimensional linear subspace that reconstructs your columns with least squared error. An autoencoder does the same job — compress each row, then reconstruct it — but the compressor (encoder) and reconstructor (decoder) are neural networks, so the "subspace" can bend. It can capture the curved, interacting structure of a policy record: that max_power, displacement, and segment move together; that certain region and vehicle combinations co-occur. Crucially it is trained unsupervised — no claim_status label anywhere. The hope is that the encoder's internal activations become a feature representation with the joint structure of the columns already baked in, which a downstream model can then exploit.

Idea 2 — denoising is what forces it to learn

There is a cheat available. Our network's hidden layers (256 wide) are wider than its input (94 dims) — it is overcomplete — so an ordinary autoencoder could simply learn the identity map: copy input to output, perfect reconstruction, nothing learned. The fix is the D in DAE: corrupt each input row, and demand the network reconstruct the clean row from the corrupted one. Now copying is useless — on the corrupted cells, the input is wrong. The only way to drive the loss down is to infer each corrupted cell from the other columns: if max_power is corrupted, recover it from displacement, segment, and model. Undoing corruption is learning the joint distribution of the columns. Formally, with corruption x̃ ~ q(·|x), the objective is:

minimize   E[ || g(f(x̃)) − x ||² ]        f = encoder, g = decoder
           reconstruct the CLEAN row x from the CORRUPTED row x̃

Swap noise — the tabular-native corruption

In images you corrupt by adding Gaussian noise or masking pixels, which is meaningful because neighboring pixels are correlated. A table has no such geometry — adding Gaussian noise to a one-hot fuel_type is nonsense. Michael Jahrer's answer (Porto Seguro, 2017) is swap noise: with probability p per cell, replace the value with the same column's value from a random other row:

x̃[i][j] = x[r][j]   with probability p   (r = a random donor row)
x̃[i][j] = x[i][j]   otherwise

The corrupted value is always a plausible value for that column — it came from a real policy — just wrong for this policy. That is exactly the corruption whose undoing teaches inter-column structure. Two study-specific routing decisions (Section 3 has the full table): the 17 is_* accessory flags are excluded from corruption (swapping a 0/1 accessory flag teaches nothing) and passed through untouched; and corruption is applied at the original-column level, then re-encoded — so a swapped categorical stays one valid one-hot, never a half-corrupted encoding (Section 6 shows the code). The swap rate p is a real hyperparameter: Jahrer used 0.15, and experiment E5 sweeps it rather than trusting him.

RankGauss — making MSE fair across columns

The reconstruction loss is a plain MSE over all encoded dimensions, so column scales matter: a raw region_density in the tens of thousands would dominate the loss while binary flags vanish. RankGauss fixes this by mapping each numeric column to a standard Gaussian by rank (sklearn's QuantileTransformer(output_distribution="normal")): the smallest value lands in the left tail, the median at 0, the largest in the right tail. Every numeric column then contributes to the MSE on equal terms. This detail is load-bearing — it is also, notably, most of what the supervised MLP in E2 needed to shine.

The deep-stack representation — what we actually hand downstream

After training, the decoder is thrown away. The feature handed to the downstream model is the deep stack: the concatenation of all three hidden-layer activations (3 × 256 = 768 dims), not just the last layer. Jahrer's observation was that earlier layers keep information the deepest layer has already abstracted away; stacking them beats any single layer. So the full pipeline is:

94-dim encoding:  21 RankGauss numerics  ⊕  17 is_* passthrough  ⊕  56 one-hot cat dims
encoder:          94 → 256 → 256 → 256   (ReLU), linear decoder head 256 → 94
training:         fresh swap noise each epoch (p = 0.15), MSE vs the clean row,
                  AdamW, early stop on a held-out reconstruction loss
feature out:      concat(h1, h2, h3) = 768 dims  →  LightGBM / logistic probe

The Porto Seguro story — told honestly

The precedent that makes this method worth testing is uncomfortably close to our problem: Jahrer's 1st-place solution to Porto Seguro's Safe Driver Prediction is also auto-insurance claim prediction. He won with a blend of six models — one LightGBM and five neural nets — and every neural net was trained on DAE hidden activations, not raw features. Swap noise at 0.15, RankGauss inputs, deep-stack features: the exact recipe above.

But three regime differences separate his setup from ours, and all three push against the DAE here:

AxisJahrer / Porto SeguroThis study
Rows feeding the DAE~1.5M (~595k train + ~892k test)46,873 (train fold only)
Fit modetransductive — the DAE saw the test-set featuresinductive — the headline DAE never sees a validation row (the fairness rule)
Regimelarge-data, where representation learning pays58k weak-signal — squarely where trees still win (Grinsztajn 2022)

The transductive point deserves a beat, because it is an integrity issue as much as a technical one: fitting the DAE on train+test features quietly uses the test distribution. Fine for a Kaggle leaderboard; not fine as a headline comparison against inductive baselines. This study's DAE is fit on the train fold only, with the fitted row count logged as a canary (n_fit_rows=46873), and the Jahrer-style transductive variant exists in the code only as a clearly-labeled aside. Representation learning's edge grows with unlabeled scale; at ~25× fewer rows and an honest inductive constraint, the method card's written prediction was that the DAE would not clear the tuned GBDT — and that a measured "no" would itself be the valuable answer. Section 4 shows what actually happened, including the result that made the question sharper than anyone planned.
