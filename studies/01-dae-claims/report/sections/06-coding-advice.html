Model Coding Advice

This is the section that makes the study useful on Monday morning. The study's centerpiece code is dae.py — a ~380-line, sklearn-style SwapNoiseDAE — plus the two-stage launcher pattern in train.py that every torch+LightGBM experiment had to adopt. This walkthrough annotates the four blocks worth stealing, then the pitfalls that bit (or tried to bite) this study.

Block 1 — swap noise at the original-column level

The subtle design decision in the whole file. Corruption happens on the dataframe, per original column, and the corrupted frame is then re-encoded — never on the encoded matrix:

def apply_swap_noise(X, eligible_cols, *, rate, rng):
    """Swap-noise corruption at the ORIGINAL-column level."""
    Xc = X.copy()
    n = len(X)
    mask = np.zeros((n, len(eligible_cols)), dtype=bool)
    for j, col in enumerate(eligible_cols):       # is_* flags are NOT in eligible_cols
        flagged = rng.random(n) < rate            # each row corrupted w.p. rate
        mask[:, j] = flagged
        idx = np.flatnonzero(flagged)
        if idx.size:
            donors = rng.integers(0, n, size=idx.size)   # random donor rows
            col_vals = X[col].to_numpy()
            Xc.iloc[idx, Xc.columns.get_loc(col)] = col_vals[donors]
    return Xc, mask

Why it matters: corrupt the encoded matrix instead and a swapped categorical becomes a physically impossible half-corrupted one-hot (two hot bits, or none) — the network learns to denoise encoding artifacts, not policy structure. Original-column corruption guarantees every corrupted row is a valid row: a swapped fuel_type is still exactly one fuel type, a swapped numeric passes through the same RankGauss map as everything else. (Because both encoders are per-column deterministic maps, this is provably identical to copying the donor's encoded slice — the dataframe version is simply the honest, testable statement of the method.)

Block 2 — the training loop: fresh noise per epoch, index-shuffle batching

for epoch in range(self.max_epochs):
    # fresh swap noise EACH epoch (proper denoising training), then re-encode
    X_corrupt, _ = apply_swap_noise(X_tr, self.eligible_cols_,
                                    rate=self.swap_rate, rng=noise_rng)
    Z_corrupt = torch.as_tensor(self._encode(X_corrupt), device=self.device_)

    self.net_.train()
    for b in kleinlib.torch_loop.iterate_minibatches(   # MPS-safe: NEVER a DataLoader
        n_tr, self.batch_size, shuffle=True, generator=noise_rng
    ):
        bt = torch.as_tensor(b, device=self.device_)
        optim.zero_grad()
        recon = self.net_(Z_corrupt[bt])
        loss = loss_fn(recon, Z_tr_clean[bt])           # reconstruct the CLEAN row
        loss.backward()
        optim.step()

    # early stop on a FIXED held-out corrupted set — a stable signal across epochs
    with torch.no_grad():
        es_loss = float(loss_fn(self.net_(Z_es_corrupt), Z_es_clean).cpu())

Three deliberate choices: (1) fresh noise every epoch — the network must never memorize one corruption pattern; (2) index-shuffle batching via kleinlib.torch_loop, never a DataLoader/TensorDataset — on Apple MPS the obvious DataLoader route silently collapses predictions to a near-constant (the campaign's war story #2, and the reason min_proba_std is a hard guard); (3) the early-stop set is corrupted once, with a fixed seed — re-noising it each epoch would make the stopping signal jitter with the noise draw instead of tracking the model.

Block 3 — inductive vs transductive, made explicit and auditable

dae = SwapNoiseDAE(swap_rate=0.15)            # fit_mode="inductive" is the DEFAULT
rep_tr = dae.fit_transform(X_tr)              # encoder fit on TRAIN fold only
rep_va = dae.transform(X_va)                  # val encoded, never seen at fit
assert dae.n_fit_rows_ == len(X_tr)           # the fairness canary: 46,873 here

# Kaggle-style aside — how Porto Seguro was actually won. NOT the headline:
dae = SwapNoiseDAE(swap_rate=0.15, fit_mode="transductive")
dae.fit(X_tr, X_transductive=X_va)            # encoder sees val FEATURES (no labels)

The API makes the integrity decision a named, greppable parameter instead of a silent concatenation, and n_fit_rows_ gives every ledger row a one-number audit trail (E3/E4 log dae_n_fit_rows=46873). If a DAE result ever looks too good, this is the first number to check.

Block 4 — two-stage process isolation (the libomp fix), actual code

The pattern that every torch+LightGBM experiment reused, from the committed E7 train.py. One file, two processes: main() re-invokes itself as a torch-only child, then runs the LightGBM head with lightgbm imported first:

def stage_impute() -> None:
    """CHILD (torch world; no lightgbm): fit/apply the DAE, dump .pkl caches."""
    import joblib
    m = joblib.load(DAE_CACHE)          # torch net inside; lightgbm NEVER imported here
    ...                                 # DAE work: reconstruct / transform / diagnostics
    joblib.dump(out, E7_CACHE)

def stage_head(t0: float) -> None:
    """PARENT: lightgbm world."""
    from lightgbm import LGBMClassifier, early_stopping   # MUST be first (war story)
    import kleinlib     # binds torch passively; no torch op ever runs in this process
    ...                 # LGBM head on the cached CPU-numpy reps + canonical evaluate

def main() -> None:
    t0 = time.time()
    r = subprocess.run([sys.executable, str(_HERE / "train.py"), "--stage", "impute"])
    if r.returncode != 0:
        raise RuntimeError(f"impute stage subprocess failed with rc={r.returncode}")
    stage_head(t0)

Rules of the pattern: the child never imports lightgbm; the parent imports lightgbm before anything torch-adjacent and never executes a torch op; everything crossing the boundary is CPU numpy in .pkl caches; and the child's return code is checked loudly. Run it under set -o pipefail with PYTHONUNBUFFERED=1, or a segfault will eat both the exit code (tee masks it) and the log (block-buffered stdout dies with the process). Cost of the pattern: one subprocess call. Proof it preserves determinism: E5's winner trial reproduced E3 bit-exactly (0.668271).

The pitfalls list

  • Never mix torch and LightGBM in one process on macOS arm64. Dual bundled libomp → SIGSEGV below Python; guards cannot catch it; import order only picks the victim. Two-stage isolate (Block 4). Detection recipe if you suspect it: (A) run the GBDT alone on cached inputs, (B) add one tiny torch op first, (C) flip import order — if A passes and B dies, you have it.
  • Never a DataLoader/TensorDataset on MPS. Index-shuffle batching (Block 2) plus a min_proba_std hard guard that raises on collapsed predictions. A collapsed model must crash loudly, not report a plausible-mediocre AUC quietly.
  • Corrupt original columns, not encoded ones (Block 1) — else you train the net to fix impossible encodings.
  • Sweep the swap rate. The "it's flat" assumption cost up to −0.0063 in this study's sweep (E5). Three points and a sidecar TSV is ~100 seconds of compute.
  • Log the inductive canary. Record n_fit_rows_ with every DAE result; it is the difference between a defensible number and a quiet transductive leak.
  • Run the supervised floor first. E2's 12-second MLP re-set this study's entire bar before the DAE ran. An SSL pipeline evaluated only against a linear baseline will look better than it is.
  • cw=None, calibrate after — for NNs too. Every class_weight=None arm here posted Brier ≈ 0.059; the lone balanced run (E1, anchor duty) posted 0.240. The doctrine transfers from GLMs to deep nets unchanged.
  • Pipe hygiene on every tee'd run: set -o pipefail and PYTHONUNBUFFERED=1. Without both, a hard crash leaves you a zero exit code and an empty log — the most expensive kind of nothing.
