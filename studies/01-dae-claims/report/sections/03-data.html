The Data

Same book as study 00: the Kaggle litvinenko630/insurance-claims dataset — 58,592 policies, 45 prepared columns, binary target claim_status at 6.40% positive (3,748 claims), zero missing values, prepared by a prepare.py byte-identical to study 00's. The data gate's verdict was GO with no blockers; what is new in this study's data card is the DAE-specific column routing — because for an autoencoder, how each column enters the network is a modeling decision, not plumbing.

The 94-dimension routing table

dae.py routes every prepared feature column into exactly one of three encoders, for a measured input dimension of d = 94:

RouteColumnsEncoderDimsSwap-noise eligible?
RankGauss numerics 21 — all numerics except is_*: continuous (subscription_length, customer_age, region_density, dimensions, power/torque…), low-cardinality ordinals (airbags, ncap_rating…), and the 0/1 binary maps median-impute → QuantileTransformer(output_distribution="normal") 21 Yes
is_* passthrough 17 accessory/safety flags (is_esc, is_parking_sensors, is_brake_assist…) passed through as 0/1 17 No — deliberately excluded
One-hot categoricals 6 — region_code (22), model (11), engine_type (11), segment (6), fuel_type (3), steering_type (3) mode-impute → OneHotEncoder(min_frequency=20), dense 56 Yes (whole column swapped, then re-encoded)

Why the is_* flags are excluded from noise

Swap noise teaches by forcing the network to infer a corrupted cell from its neighbors. That only produces useful gradients when the cell has structure worth inferring. Swapping a 0/1 accessory flag mostly toggles a bit whose recovery signal is thin — a coin-flip-shaped pretext that adds loss noise without teaching joint structure. So the 17 flags ride along as clean context (the network may use them to reconstruct other columns) but are never themselves corrupted. The same 27 eligible columns (21 numerics + 6 categoricals) later define where E7 injects MCAR missingness — the imputation experiment deliberately mirrors the denoising doctrine.

The redundancy that becomes an asset

The data card's most interesting note: seven columns are near-deterministic functions of model (engine_type, displacement, cylinder, and the four torque/power columns) — 11 vehicle models, at most 11 distinct values each. For a GLM that is collinearity to manage; study 00 dropped them for its tree recipe. For a DAE this redundancy is exactly the inter-column structure the denoiser feeds on — a corrupted max_power is recoverable from model almost by lookup. The card kept all seven, and two experiments later confirm both faces of the coin: the redundancy is why the DAE imputes categoricals at 90% accuracy (E7) — and why handing trees the same information twice, raw plus re-encoded, actively hurts (E6).

Inherited guardrails, verified live

Two disciplines carried from study 00, both confirmed against this environment rather than assumed. First, the value-pattern check: under this machine's pandas 3.0.3, the 19 boolean-ish string columns report dtype str, not object, so a naive dtype == "object" gate would silently skip all 19 — prepare.py detects them by their actual value sets ({"Yes","No"}) and maps them deterministically. Second, the 6.4% imbalance is handled by the calibration doctrine, not re-weighting: every downstream classifier here trains with class_weight=None (E1's balanced setting exists only to reproduce the campaign anchor). The DAE itself is unsupervised — imbalance cannot bite it; it bites only the classifiers on top.

Row, column, and target counts match the ancestor campaign exactly — and the E1 anchor then proved it bit-exactly: val_auc 0.625462 vs the campaign's 0.6255, a gap of 0.000038. The split is the same split; every comparison downstream is legitimate. Full audit: data_card.md.
