Model Coding Advice

This is the section that makes the study useful on Monday morning. The centerpiece code is two library-stable files — generator.py (the known-truth lab) and estimators.py (the four estimators) — plus a deliberately thin train.py that swaps one cell per experiment. Here are the four blocks worth stealing, then the pitfalls that shaped every finding above.

Block 1 — the truncated + censored likelihood

The proper conditional likelihood is what a naive MLE throws away. Uncensored points are conditioned on being above the deductible, so each contributes f(x)/S(d); censored points (recorded at the limit u, flagged) contribute S(u)/S(d); anything recorded at or below d can only be contamination and is dropped as unobservable under the truncated model:

def nll(theta):
    dist   = _frozen(family, theta)
    log_sd = np.log(dist.sf(d)) if d > 0 else 0.0    # log S(d): the truncation normalizer
    ll     = np.sum(dist.logpdf(unc)) - len(unc) * log_sd     # uncensored: f(x)/S(d)
    if n_cens:
        ll += n_cens * (np.log(max(dist.sf(u), 1e-300)) - log_sd)   # censored: S(u)/S(d)
    return -ll

res = minimize(nll, theta0, method="Nelder-Mead",
               options={"xatol": 1e-6, "fatol": 1e-8, "maxiter": 4000})

This is the estimator that won the realistic cell (E5, 7.85%). Its cost is speed: Nelder-Mead on this likelihood took 13.6 s of E4's 20.7 s wall-clock — the one place compute mattered. Note the 1e-300 floor inside the log: without it, a parameter draw that pushes S(u) to underflow crashes the optimizer mid-search.

Block 2 — the observable-window quantile grid (the QLS core)

Two branches. When the data is complete, QLS is the three-line closed form (a polyfit whose slope is σ̂). When a deductible or limit is present, match the empirical quantiles to the conditional model quantiles over the known window — the same identifiability trick from Section 2, in code:

qhat = np.quantile(z, p)                       # empirical quantiles of log-loss
full_line = (not isfinite(a)) and (not isfinite(b))   # a = log d, b = log u

if full_line and weights == "ols":            # complete data: the closed form
    zp = norm.ppf(p)
    slope, intercept = np.polyfit(zp, qhat, 1)
    return Fit("lognormal", {"mu": intercept, "sigma": abs(slope)})

def cond_q(mu, sigma):                         # windowed: CONDITIONAL model quantiles
    lo = 0.0 if not isfinite(a) else norm.cdf((a - mu) / sigma)   # F(d)
    hi = 1.0 if not isfinite(b) else norm.cdf((b - mu) / sigma)   # F(u)
    return mu + sigma * norm.ppf(lo + p * (hi - lo))
# fit: least_squares(lambda th: sw * (qhat - cond_q(th[0], exp(th[1]))), x0)

The lo + p*(hi−lo) re-maps each grid point onto the observable slice [F(d), F(u)] — the whole reason window-QLS recovers the truth (E4, 4.83%) where a naive fit bends off the diagonal. The obs = x[(~cens) & (x > d) & (x < u)] filter is the robustness: unit typos (×0.01) fall below d and gross errors (×10–100) mostly above u, so the known policy terms screen the contamination for free.

Block 3 — the GLS weights (and why they disappointed)

The efficient inverse-variance weights, from the asymptotic covariance of sample quantiles, plugged in from a first-stage OLS fit:

if weights == "gls":
    mu0, s0 = z.mean(), z.std(ddof=0)                  # first-stage OLS-ish fit
    q0   = mu0 + s0 * norm.ppf(clip(p, 1e-4, 1 - 1e-4))
    dens = norm.pdf((q0 - mu0) / s0) / s0              # f(Q_i) at the grid
    w    = dens**2 * n / (p * (1.0 - p))               # Σ⁻¹ diagonal (inverse variance)

The theory says this should beat OLS. It did not (E2: 1.123× the MLE error vs OLS's 1.083×). The plug-in of f and Q from a noisy first-stage fit injects more variance than the weighting removes on a 19-point grid; the full off-diagonal Σ−1 is the true efficiency ceiling but ill-conditions on fine grids. Practical rule: default to OLS. Only reach for full-covariance GLS on a coarse grid where the covariance inverts stably.

Block 4 — the premium functional (truth and fit, scored identically)

The single most important design choice in the lab: truth and every estimate are priced through the same routine, so a difference is never an artifact of two code paths. It is the stop-loss integral of the survival function, finite because the limit u caps the interval:

def layer_functionals(sf, qf, d, u, loading=0.10):
    mean_payout = quad(sf, d, u, limit=200)[0]              # E[Y] = ∫_d^u S(x) dx
    var99  = min(qf(0.99), u) - d if 0.99 > F(d) else 0.0
    tail   = quad(sf, d + var99, u, limit=200)[0]
    tvar99 = var99 + tail / (1.0 - 0.99)
    premium = mean_payout + loading * (tvar99 - mean_payout)
    return {"mean_payout": ..., "var99": var99, "tvar99": tvar99, "premium": premium}

Because the layer is capped at u, even a ξ=0.4 GPD tail integrates to a finite TVaR — which is precisely why the whole fire tail was worth only +0.174% of premium (E6). A policy limit is a tail-risk control written into the integral bounds.

The pitfalls list

  • Do not use the diagonal plug-in GLS. It added noise, not efficiency (E2: 1.123× > OLS 1.083×). Default to OLS; reserve full-Σ GLS for coarse grids.
  • Do not trim blind. On a mixture, the upper tail can be real risk, not contamination — trimming it deletes signal (E5: trim-only 35.90% vs window-only 8.90%). Let the observable (d, u) window do the screening; trim only what the window cannot reach.
  • Never evaluate on parameter metrics alone. E4's naive MLE had huge μ̂/σ̂ RMSE (0.60 / 0.35) yet a −5.2% premium error — the biases cancel in the layer map. Always push the fit through to the premium/VaR/TVaR you actually sell.
  • Model the mixture before blaming the estimator. Every single-family fit underprices this book by 23–42% at ε=0 (E6, E7) — misspecification, not estimation. A robust estimator cannot rescue the wrong family.
  • Guard the synthetic truth, and report against the MC floor. Gate the generator against its own truth before any experiment (E1: bias < 2·SE), and score every result against the 4.28% floor, never against 0 — or you will chase sampling noise as if it were bias.
  • Watch the contamination direction. This mix biases premiums up (overcharge); truncation-ignorance biases them down. Knowing the sign is half of reading the decision table.
